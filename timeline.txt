2026 — RAG Systems, Agentic AI & Building PartnerAI

Completed the IBM RAG and Agentic AI Specialization through IBM on Coursera.

This specialization gave me a much deeper understanding of retrieval-augmented generation (RAG), agentic workflows, orchestration patterns, and how modern AI systems are actually engineered beyond just prompting an LLM.

Alongside the specialization, I built a project called PartnerAI.

The idea was simple:
upload business documents (reports, datasets, PDFs, etc.) and interact with them naturally instead of manually digging through files.

Rather than hallucinating, the system retrieves relevant context directly from uploaded data before generating responses. It could also generate charts dynamically for quick analysis and decision-making.

Some interesting parts under the hood:

Multi-agent architecture using CrewAI with role-based agents like a Business Analyst and Data Scientist
RAG pipeline powered by ChromaDB for contextual retrieval
Async backend built with FastAPI
Python REPL execution tools allowing agents to run Python code dynamically on datasets
React + Vite frontend

This project was one of the moments where multiple areas I had explored earlier — backend systems, AI, automation, data analysis, and architecture thinking — finally started connecting together into a single system.
